# Code of Conduct

This project is small, safety-focused, and intended for public
collaboration. Contributors and maintainers are expected to keep
discussions respectful, factual, and useful.

## Expected Behavior

- Be direct and civil.
- Assume good intent, but correct unsafe guidance promptly.
- Keep reports sanitized and reproducible.
- Focus disagreement on behavior, documentation, tests, or safety impact.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Posting tokens, credentials, private keys, customer data, private
  repository identifiers, or raw secret-bearing logs.
- Presenting assumed or fabricated GitHub/agent behavior as measured
  fact.
- Repeatedly ignoring maintainer requests to sanitize sensitive material.

## Enforcement

Maintainers may edit, hide, or delete unsafe public content; close issues
that cannot be sanitized; or restrict participation for repeated harmful
behavior.

Security-sensitive material should be reported privately through GitHub
security advisories instead of public issues.
